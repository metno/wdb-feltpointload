Introduction

This document describes the specific tests to be carried out during the system test of the FeltLoad component, used for the FELT Field loading. The purpose of this component is to load meteorological fields into the WDB system.

Purpose

The test specification should:

  • Clarify the functionality/performance/accesibility requirements of the system

  • Clearly specify which tests must be passed for the component to be accepted for production

  • Document the consensus for the level of testing of the WDB

The tests set forth in this document are designed to satisfy the demands of two distinct stakeholders in the product; namely the end-users of the WDB and the operational maintenance department.

For each test, it must be defined who has the responsibility for performing the test. Any limitations with respect to the test must also be specified (e.g., for example if the test has yet to be implemented).

Controlling Document

The test specification and implemented tests in the FeltLoad component follow the principles described in the WDB Test plan [1].

Compilation Test

There should be no warnings during the compilation or linking of the FeltLoad component.

Unit Tests

There are no unit tests for the FeltLoad component.

Installation Tests

Installation tests examine that the functionality of the system fulfills the system requirements of the system as defined in Bugzilla. We employ the Category-Partition test design pattern [2].
